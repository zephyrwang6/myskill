# 产品战略分析框架

## 一、创意理解与分类

### 核心价值提取
- **问题定义**：这个产品解决什么问题？
- **目标用户**：谁会使用这个产品？
- **核心功能**：产品的核心价值主张是什么？

### 赛道定位
- **行业分类**：属于哪个行业/细分领域？
- **竞争格局**：现有竞争者有哪些？市场结构如何？

## 二、倒推法分析（从终局往回推）

### Step 1: 终局推演（5-10年后）

**思考维度：**
- 这个领域的终极形态是什么？
- 技术发展到极致会是什么样？
- 用户需求的终极满足方式是什么？
- 这个产品在终局中还存在吗？还是会被什么取代？

**输出格式：**

```
📍 2030-2035终局:
- [描述终局场景]
- [关键变化]
- [产品形态演变]

❓ 关键问题: 这是真需求还是过渡性需求？
```

### Step 2: 路径倒推（必经之路）

从终局往回推，识别关键节点：

```
2030-2035: [终局状态]
    ↓ 需要什么条件？
2027-2030: [前一阶段]
    ↓ 需要什么条件？
2026-2027: [再前一阶段]
    ↓ 需要什么条件？
2025: ← 当前位置
```

**输出格式：**

```
🔄 倒推路径:
- 每个阶段的关键特征
- 阶段转换的触发条件
- 技术/市场/政策的依赖
```

### Step 3: 价值转移分析

**思考：**
- 在演进过程中，价值会从哪里转移到哪里？
- 什么会消失？什么会留下？
- 什么是绕不过去的关键环节？

**输出格式：**

```
💰 价值转移图:
- 当前价值点: [X]
- 未来价值点: [Y]
- 转移路径: [X → A → B → Y]
- 持久价值: [不会被替代的部分]
```

## 三、顺推法分析（从现状往前推）

### Step 1: 现状评估

- **技术成熟度**：当前技术能否支撑？
- **资源能力**：现有资源和能力有哪些？
- **市场教育**：用户是否理解这个产品？
- **基础设施**：必要的基础设施是否就绪？

**输出格式：**

```
📊 现状扫描:
✅ 已具备: [列举]
⚠️ 部分具备: [列举]
❌ 尚未具备: [列举]
```

### Step 2: 渐进式演进路径

```
第一步(0-6个月): [MVP]
    ↓ 验证什么？
第二步(6-12个月): [迭代方向]
    ↓ 验证什么？
第三步(1-2年): [规模化]
    ↓ 验证什么？
```

**输出格式：**

```
🚀 顺推路线:
- 每个阶段的里程碑
- 需要验证的假设
- 资源需求
- 风险点
```

### Step 3: 快速验证点

- **MVP定义**：最小可行产品是什么？
- **验证方法**：如何用最低成本验证核心假设？
- **试错机制**：哪些是可以快速试错的？

## 四、双向验证（倒推+顺推的交叉分析）

### 1. 时间窗口分析

- **倒推视角**：这个机会什么时候成熟？
- **顺推视角**：我们什么时候能做到？
- **差距分析**：是太早（市场未成熟）还是太晚（已被占领）？

### 2. 能力匹配度

- **能力要求**：倒推出的关键能力要求
- **能力基础**：顺推出的现有能力基础
- **差距识别**：差距在哪里？如何弥补？

### 3. 机会评估矩阵

```
        高壁垒
          ↑
          |
          |
          |
短周期 ←——+——→ 长周期
          |
          |
          |
          ↓
        低壁垒
```
