# 写作风格参考

## 参考书籍：《人人都是产品经理》苏杰

### 写作风格特点

1. **案例驱动**
   - 用真实工作经历中的案例说明概念
   - 先讲故事/场景，再提炼方法论
   - 案例要具体到细节，让读者有代入感

2. **系统化与结构化**
   - 每章有清晰的逻辑框架
   - 围绕核心问题展开：为什么做、做什么、怎么做、和谁做
   - 章节之间有递进关系

3. **实用与可操作性**
   - 提供可落地的工具/框架/模板
   - 避免空谈理论，要能"拿来就用"
   - 给出具体步骤和检查清单

4. **循序渐进**
   - 从简单到复杂，从入门到进阶
   - 每个概念先给定义，再给例子，最后给方法

5. **口语化表达**
   - 像朋友聊天，不像教科书
   - 多用"你"、"我"、"我们"
   - 适当使用比喻和类比

### 章节结构模板

```
1. 开头：场景/痛点引入（50-100字）
2. 核心概念定义（100-200字）
3. 为什么重要（100-200字）
4. 怎么做（详细步骤 + 案例）
   - 步骤1 + 案例
   - 步骤2 + 案例
   - ...
5. 常见问题/避坑指南
6. 总结 + 金句
7. 行动清单（可选）
```

### 写作原则

- **详略得当**：核心内容详写，辅助内容略写
- **有血有肉**：抽象概念要配具体案例
- **金句点睛**：每章有 1-2 句可以划线的金句
- **读者视角**：始终问"读者看到这里会有什么疑问"

## 知识管理方法论参考

### 经典框架

- **DIKW 模型**：数据 → 信息 → 知识 → 智慧
- **费曼学习法**：学习 → 教授 → 反馈 → 简化
- **卡片笔记法**：原子化 → 关联 → 涌现
- **GTD**：收集 → 处理 → 组织 → 回顾 → 执行

### 可引用金句

- "知识不是力量，应用知识才是力量。" —— 培根
- "最好的学习是教会别人。" —— 费曼
- "工具是死的，系统是活的。"
- "输入不等于吸收，输出才是真正的学习。"
- "你不是缺工具，你缺的是系统。"

## 项目管理方法论参考

### 经典框架

- **SMART 原则**：具体、可衡量、可达成、相关性、时限性
- **OKR**：目标 + 关键结果
- **看板法**：可视化 + 限制在制品 + 管理流动
- **敏捷迭代**：小步快跑、快速反馈、持续改进

### 可引用金句

- "计划赶不上变化，但没有计划的变化是混乱。"
- "目标不是用来达成的，是用来指引方向的。"
- "完成比完美更重要。"
- "如果一件事值得做，就值得做得糟糕。" —— G.K. Chesterton
