## 参考文章 1：Claude Code 与软件开发变革

下午重新续费了 Claude，为了更好的用 Claude Code。最近的感觉是，Claude Code 还在快速破圈。X 上关注的 AI 创业者，对它的评价非常高。

Opus 4.5 模型 + Skills，让我看到新一代 Agent 的样子。

刚刚又刷到一条 X，原贴我简单翻译下：

Claude Code 搭配 Opus 4.5，是一个真正的分水岭时刻，它正在把软件开发这件事，从手工、工匠式的创作，推进到真正的工业化流程。

它就像活字印刷术。就像缝纫机。就像照相机。

然后 Karpathy 回复说，他也有类似的感受。

如果大家最近用过这个组合，应该能理解他们的震惊，绝对不是咋咋呼呼。软件开发这个行当，离变天确实不远了。

一个好工程师，像个手艺人，靠经验、感觉、长期训练。而现在，它开始变成工业流程，可以拆分、可以复制、可以规模化。

之前，知名程序员 DHH 曾经公开反对过 AI 写代码这件事。因为他觉得 AI 写代码的过程中，工程师很难从中学会这门手艺。当程序员不再思考语法和代码细节时，能力也会随之衰退。

但昨天，他承认，自己最近也开始使用 AI 写代码了。AI 可以快速生成初稿，提高效率。

说实话，我能理解 DHH 之前的心态。因为这哥们是天才型程序员，所以，当 AI 也能写代码的时候，他作为工匠型的人才，内心肯定是排斥的。

对他来说，写代码不是简单的把需求变成结果，而是一种手脑并用的思考过程。怎么拆解问题、怎么组织代码结构、怎么设计函数，这些本身就是手艺。

他很难从旧的、自己擅长的世界里走出来。

还好。DHH 转向很快，现在他全盘接受了这件事。再不接受，可能真的就成中登了。时代在变，守着旧的手艺也没问题，只是会越来越边缘化。就像现在也还有人喜欢黑胶唱片一样。

Coding Agent 这事，今年肯定会成了。

前年，我们兴奋于 Cursor 多维度的自动补全，去年，我们兴奋于 Agent 可以独立完成一个相对复杂的功能。那今年，我们会被 Agent 彻底折服。

这是我的感觉。

刚刚，我们公众号一个铁粉给我发了三个文件，让我判断哪篇是我写的。我虽然知道那都是 AI 的作品，但仍然非常惊慌。

因为它基于 Claude Code 的 Skills 做了一套流程，喂了我最近的一些文章，然后可以快速生成我文风的内容。

我突然在想，那我接下来的竞争力是什么？

观点？感受？人味？审美？不知道。我没有得到答案。独自迷茫在华灯初上的夜晚。

## 参考文章 2：为什么我们讨厌AI味的内容

这两天有点摸鱼，因为我终于把《怪奇物语》第五季给补完了。说实话，最后大战有点仓促，但不妨碍它依然是一部好剧。看完之后，我意犹未尽，像往常一样，打开B站，想找个解说视频，看看别人的分析，看看有没有我遗漏掉的细节，补一下整体世界观。然后我就点开了一个播放量很高的万字拆解视频，这个UP主出的超级快，基本属于剧集更新之后，他一天出一个万字拆解的40分钟视频。在开开心心的点开看了之后，我感受到了暴击，这视频看的我究极痛苦，看了20多分钟，就关掉了。真不是他做的拆解做的不好，也不是他的声音难听或者画面不行。主要原因，在于他的文案。

不知道大家平时会不会看到这样的文案，我随手列几句，大家估计就会有一种很熟悉的感觉了。。。

比如这一句："就不只是"末日摇滚"的情绪匹配，而是"广播室的报警""

我当时听到这一句，真的一皱眉摸了摸脑袋。

还有：
- 能把她拉出来的不是道理，而是一个能让情绪重新对的节拍和旋律。
- 她不是靠蛮力去赢，而是靠在崩溃边缘依旧抓住"我还想活"的那一点点意志。
- 他的退一步不是不爱，而是忠于尊重南希的选择，他需要的不是被保护或者被安放到某个关系里，而是被允许继续向前冲，继续选择自己的人生路径。
- 南希"选择"的也不是一种绑定，而是对这种坦诚的认可。

如果觉得看文案不出戏的，可以自己找个地方，然后把开着录音，把这段文案读出来，再回放给自己听，就能感受到有多奇怪了。

真的，作为一个每天看无数AI生成内容的一个人，我现在看到这样的文案，真的有点头皮发麻和生理性反胃。弹幕里其实也有不少朋友评论。我无意去抨击UP主用AI生成内容啥的，因为这个很正常，毕竟时间窗口就那么短，用AI辅助一定是必然的，才能以最快速度把内容端到粉丝的面前。

我是当时在思考，不止是我，为什么现在，大家这么的讨厌，这种"不是...而是..."的句式。为什么一个原本再普通不过的句式，会在短短一两年里，迅速变成很多人厌恶的东西？为什么我们会开始下意识地讨厌AI生成的内容，甚至连人类写作，只要沾上一点相似的味道，也会被我们误伤？

我最近一直在想这个问题。

坦率的讲，在几年前，"不是...而是..."这种句式，还是标准的写作范本。"不是A，而是B"，这个句式在逻辑学上，叫做"否定肯定式"，它本身是一个非常好的工具，用以强调重点，制造认知反差。

比如这句话：我爱上的，不是你的容貌，而是你的灵魂。

你看，一句话，基本就构建出深度和诗意。

问题是，这玩意，是个模板，是个套路。当一个套路被滥用的时候，它就成了一种枷锁。

闻一多《宫体诗的自赎》里面有个词，叫，陈词滥调。大概意思就是，一些表达方法、概念、或是艺术相关作品具备的元素，因为过于频繁地被使用，以至于丧失了它原本的意义或原先想达成的效果，甚至于到达毫无新意、无趣而且恼人的程度。

"你以为A，其实B"的结构，一开始确实是个好用的工具。用得好，可以给人一种视角被翻转的爽感，也能让读者觉得，你这小子还是有点东西还是挺深刻的。

但是这玩意，AI实在是用的太多了。

大模型写东西的方式，在你不给任何Prompt输入和约束的情况下，就让它干巴巴写的写一个XXXX。本质上，就是从海量语料里，总结出哪种写法出现得最多，然后在你给的提示下，倾向性地选择那些最稳妥的组合。

就这简单的几句话，滥用双引号，不是而是的句式，雷直接踩遍了。甚至不止句式，你让它给你写几个可以证明这个观点的案例，你会发现，奶奶的GPT、Gemini、Claude啥的，都会把那几个重复案例翻来覆去的讲。

再结合着现在的内容工厂，一天可能就有上亿的AI生成的文本内容，被扔到互联网上。里面充斥着大量的，没有人类修改过的、粗糙烂制的AI内容，也大量的使用着，这种"不是...而是..."的句式。

而这些内容，并不像曾经作者写的内容一样，会斟酌会谨慎会思考，而是对用这种句式，形成了一种肌肉记忆。因为只要用上，就能显得我牛逼，显得我深刻，那我怎么可能不用嘛。

于是，一篇篇的内容，变成了新时代的八股文。

明清时候的八股文，就是把文章的结构、韵律、甚至用词，都用一套严苛的格式给框死。考生不需要有天马行空的思想，只需要把自己的观点，像填空一样，塞进"破题、承题、起讲、起股、中股、后股、束股、大结"这个固定的模子里，就足够了。

你不需要创新，你也不能有自己的结构，格式严格，限定字数，不许违背经注，不能自由发挥。

后来大家终于觉得有些不对，康熙帝在1663年说，"这玩意空疏无用，实于政事无涉"，于是宣布废除，结果废的不彻底，1665年又被当时的礼部侍郎上疏请复，结果又恢复了。直到最后，光绪二十八年，也就是1902年，八股文终于迎来大结局，清政府宣布停止科举考试使用八股。1904年，清政府举行了最后一次科举，翌年起废除。

在整个历史上，骂的最狠的一哥们，叫顾炎武，他是这么说的："八股之害等于焚书，而败坏人才，有甚于咸阳之郊，所坑者但四百六十余人也。"

大概意思就是说，八股文的祸害，跟焚书差不多，它把人才带坏、把读书人的路子毁掉，这种伤害甚至比秦始皇在咸阳郊外坑杀儒生还更严重。因为当年被坑的也就四百六十多人，八股文害的却远远不止这么一点人。

当然，我的意思不是说这年头，用AI辅助生产内容，就是这个时代的八股文，这个差别还是挺大的，而是没有自己的观点，纯靠AI自动化生成内容，扔到互联网上，那些内容，跟八股文其实也差不多。

回到最初的问题上，我们到底在讨厌什么？我们真的讨厌的是这个句式吗？

其实在我看来并不是，我们讨厌的，是模板化的套路八股文，还有，一种预制内容的荒谬。

前几个月，预制菜这事打的有多猛，相信大家都围观了全程。其实很多人不喜欢预制菜预制菜的理由，和现在越来越多人本能抗拒AI味的内容，还挺像的。

预制菜这玩意，是好吃又省事的工业化复制品。AI生产的八股文内容，是好看又省事的工业化内容。

大家不喜欢预制菜，很多时候不是接受不了预制菜，就像老罗当时一直的论点，你可以用，但是你得跟我说你用了，你不能说你用了预制菜但是结果你跟我说没用对吧。

我们常常看到，预制菜最惹人恼火的场景，都发生在那些打着"现炒现做""妈妈味道""农家小院"的招牌，却端出一堆预制菜的店。

AI内容最让人反胃的时刻，也基本类似，就是你点开一个是用一个真人账号发出来的，看上去挺用心的B站长视频、知乎长文、公众号深度好文，我们心里的预设一般都是："这是他认真写给我们的东西。"

开头还挺期待，读着读着，整个人突然一激灵。艹，怎么满眼都是那种模板句式，那种看到了无数遍的结构，那种中立客观到没有一点棱角的表达。

不是，这哥们用AI写的？？？

然后你就生气了。

更多时候，我们心里那句骂人的话，其实是："你怎么不提前说一声，这玩意是预制的？"

虽然一个是菜，一个是内容，但是都是给人消费的，也没啥区别。

这跟全球范围内对AI内容的不信任是联动的。之前纽约时报在25年4月份，搞了个针对2000个美国人的问卷调查，说大多数人网络信任感直线下降，很大一部分原因就是AI生成内容太多，真假混杂，大家已经很难分辨自己眼前的东西到底是人写的，还是机器拼凑出来的。

他们，认为他们消费的网络内容中只有41%是完全准确、基于事实并由真实人类创建的。

对互联网内容的不信任，已经达到了历史的巅峰，以后只会继续的，越来越高。全球其实都差不太多。

其实整个的背后，就是一个老生常谈的话题。就是，我们为什么会和人交流？我们看一篇文章，听一段话，本质上，是为了什么？

其实，不就是是为了和另一个意识产生连接嘛。

我们想知道，屏幕的背后，那个打字的人，他经历了什么，他感受到了什么，他想告诉我什么。无论你是让我开心，还是让我获得知识，还是让我产生共鸣。

语言，只是一个载体，一艘船。我们真正通过信息触摸的，其实是船上那个活生生的人。

那人他划得歪歪扭扭，时不时还会撞在礁石上，然后冲着破洞缝缝补补，可你能看见那个人在流汗、在犹豫、在改道，没事还像你招招手。

但是现在越来越多的AI生成的内容，变成了那种就像带着王八壳自动行驶的集装箱，外壳巨大光滑，航线极其稳定，货物装得满满当当，可你望过去，甲板上空无一人。

你站在岸上，看着它从眼前经过，嘴里只会冒出五个字：关劳资屁事。

在一个信息爆炸，内容过载的时代，我们比任何时候都更渴望与另一个真实的灵魂连接。

我们刷短视频，看直播，读文章，我们拼命地消费内容，本质上是在寻找同类，是在确认，我不是一个人。

这明明是个信息流转的究极迅速的年代。却又好像是个孤独至死的年代。

在这种所有东西都可以预制、可以复制、可以自动生成的世界里，我和另外一个具体的人的那条连接，好像也在慢慢变没了。

我点开一个博主的视频，听到的，确是一个模型从一堆语料里算出来的最优起号话术。我看一篇所谓的事件解读，看到的，却又是写作课加AI模板算出来的标准深度。有的时候我淘宝买点东西生气了跟客服吵架，对面回过来的，也是一段经过AI优化过的安抚话术。

我尼玛。

久而久之，人真的会变得很疲惫。因为你已经不知道，这些词语背后，到底还有没有一个人在承担责任。你不知道该跟谁生气，也不知道应该把感动分给谁。

所以，我们讨厌"不是...而是..."那套句式，究竟实在讨厌什么？

我觉得，表面上，当然就是那种满屏的模板句式。往里一点，就是对于AI内容的不信任。再往底层，就是，我们不想让自己和别人的那点真实连接，被工业化的内容生产彻底磨平。

罗振宇老师在前几天结束的《时间的朋友》年度演讲里面，有一个在AI时代，怎么教育孩子的篇幅，其中有一句建议非常的好。

就是想方设法，帮孩子逃离一致性。

我觉得不止是孩子，对于我们也一样。

当你还对这些内容，产生厌恶的情感的时候。请相信你自己，你也不是一个，想被工业磨平的人。

我一直觉得，人类对"假"的敏感，很多时候比自己以为的要强。

从这个角度看，这种厌恶甚至还挺珍贵的。

希望，大家都能在这个时代里。保持自己的这种珍贵。

## 参考文章 3：High Agency - 不经允许就擅自成功的能力

这几位大家可能在生财有术的圈子里听过：盗坤、紫菜，还有damo。在和他们沟通时，我捕捉到一种非常奇妙的感觉。这种感觉和他们目前赚了多少钱、做了多大的盘子无关。而是一种更底层的信号——只需要听他们讲超过10分钟，看着他们的眼睛，听他们的语调，你的大脑就会自动产生一个判断："这哥们，无论做什么项目，哪怕现在是一片废墟，他也一定能成。"

这种感觉极其强烈，甚至带有一种压迫感。尤其是上个月，在北京的生财线下活动，又听了一场盗坤的演讲，这种感觉尤其强烈。 （可惜没有直播也没有录播，纯线下活动，否则我想推荐你多听几遍）

**那个词叫 High Agency**

著名创投机构a16z很喜欢提到一个词：High Agency。中文很难精准翻译，有人译作"高能动性"，但我更愿意把它定义为："不经允许，就擅自成功的能力"。

如果一定要强行翻译的话 —— High Agency指的是：一个人强烈相信"我能影响结果"，并且会主动采取行动去改变局面，而不是被动等待、抱怨环境或把结果完全归因于外部。

看到盗坤、紫菜和damo，我明白了，原来这个词说的就是他们。

绝大多数普通人，也就是 Low Agency 的人，他们的人生叙事是"受害者模型"：
- "因为环境不好，所以我做不成。"
- "因为平台规则变了，所以我流量下滑。"
- "因为没有技术合伙人，所以我产品上不了线。"

而在盗坤、紫菜这些人身上，你永远听不到这种句式。他们拥有极高的主体性。在他们的操作系统里，环境只是一个参数，而不是借口。如果环境是A，他们不会抱怨A，而是迅速计算："既然环境是A，那我就调高参数B，再引入变量C，最终我还是能得到结果D。"

你在那10分钟里感受到的，不是盲目的自信，而是这种"对现实的绝对掌控感"。这就是乔布斯所谓的"现实扭曲力场"的平民版。

**普通人如何修练 High Agency？**

我写这篇文章，不是为了让你去崇拜大佬。我是想号召大家：如果要成事，你得先成为这样的人。

一些小的实操：

**1. 删除垃圾表情包**

请你和人聊微信的时候，不要用这些表情，也不要和总喜欢发这种表情的人聊天。

我非常讨厌和老发这种表情的人聊天。你要丧，你自己偷偷丧你的，别来影响我心情。High Agnecy的人，更习惯发的是这样的表情。

希望你早日习惯这些新表情，把你的能量场支棱起来。

**2. 戒掉"等待许可"的心态**

不要等风口，不要等完美团队，不要等准备好了再出发。 High Agency 的核心就是：遇山开路，遇水架桥。不要说"我不会"，在这个时代，请把这句话改成："我怎么用 AI 搞定它"。

**3. 练习"改写叙事"**

从今天开始，监控你的语言。把所有的"因为……所以不行"，强制改写成"虽然……但是我能……"。这种语言模式的改变，会反向塑造你的大脑回路，把你从"受害者"变成"掌控者"。

**4. 用小胜喂养自信**

那种"必成"的气场，不是装出来的，是一次次"拿到结果"喂出来的。先定一个极小的目标，然后用雷霆手段把它做成。这种胜任感会叠加，最终形成你的气场护城河。信心是一种自证预言。

在AI时代，如你所见，技术壁垒正在被无限拉平。但这反而让High Agency变得更加稀缺。

## 参考文章 4：用 Gemini 两三个小时做了十个小应用

我用 Gemini 3 Pro 两三个小时做了十个小应用，主要用的是Gemini 的应用和Google AI studio。制作方式也很简单，选择 Gemini 里面的Canvas，描述并发送自己的需求。基本上两次对话就能生成一个像这样子的一个应用。如果是在 Google AI studio 里就选择build模式也能制作。

Gemini不仅将代码的能力提升到顶级，而且谷歌平台提供的基础设施可以免费制作并发布应用。真的财大气粗，完全不用担心token 花钱，它使用内置API，还能连接github，快速将我们想法落地，发布给用户体验。在Google AI studio的build里就可以免费用。

下面就是10个应用的制作过程，附体验地址，可以在电脑和手机浏览器里体验，相信会激发你创作产品的灵感。

**1 城市名片生成器**

第一个应用是一个城市名片生成器，输入一个城市，然后它能给我这个城市做一个地图的绘制，旅游路线、城市数据和特色美食的推荐。看这个地图上面的绘制的方向，和我们现实世界的位置是相联系的。还可以选不同的颜色，还能把这个图片下载下来，可以看到这个前端效果特别的漂亮，整个卡片也很优雅。这个制作的应用比较好的点是，它给你内置了一个的API。我不知道它这个里面用的是下面的3还是2.5，但它这个效果真的很好。

体验地址：https://gemini.google.com/share/295b5dcd6c96

**2 生日档案生成器**

这是一个生日档案的生成器，输入出生日期，然后它就调用Gemini API去获取到1999年12月11日这一天发生了什么事情。同日生日的名人，1999年这一天的大事，还有这一年的冷知识，这非常有意思，而且整个设计是用了复古的windows风格。

体验地址：https://gemini.google.com/share/ec45d9cdbdc2

**3 AI可视化生成**

这是一个将文本通过AI可视化为流程图、SVG、Html-PPT的生成器。我这里是随便搜了一个文章叫背影，它生成了一个关于这篇文章的一个逻辑路线图。样式简约美观。

体验地址：https://gemini.google.com/share/2f994ec1fe47

**4 动物生命周期卡片**

这个小应用只要用户输入一个动物，它能够根据这个动物描述这个动物的成长的一个周期，非常适合给小朋友教学。比如下面这个，我输入蝴蝶，得到了这个图：上面标题下是一个关于蝴蝶的冷知识：蝴蝶的生命周期虽短，但它们的幼虫在几周内增加到出生时的3000倍。然后下面是第一周、第四周、第八周的整个生命过程的描述。这个卡片也可以下载成PNG。制作原理，就是让AI输出SVG的语言，可视化展示整个信息。
